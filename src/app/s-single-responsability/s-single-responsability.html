S - Single Responsibility Principle (SRP)

O Single Responsibility Principle (Princípio da Responsabilidade Única) afirma que uma classe deve ter apenas uma única razão para mudar. Isso significa que ela deve ser responsável por apenas uma funcionalidade específica do sistema.

Neste exemplo, o componente UserCardComponent tem a única responsabilidade de exibir os dados do usuário na interface.

  • O serviço UserService é responsável por buscar os dados do usuário.
  • O serviço LoggerService é responsável por registrar logs no console.

Separar essas responsabilidades em classes distintas facilita a manutenção, o teste e a reutilização do código.

✨ Uma mudança na forma como buscamos usuários não afeta a lógica de exibição ou de log.
