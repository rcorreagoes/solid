O - Open/Closed Principle (OCP)

O Open/Closed Principle afirma que classes devem estar abertas para extensão, mas fechadas para modificação.

Neste exemplo, usamos a interface DiscountStrategy para definir o comportamento de estratégias de desconto. Assim, podemos adicionar novas estratégias (ex: desconto para estudantes) sem alterar o componente principal.

  • RegularDiscount aplica 10%
  • PremiumDiscount aplica 25%

O componente DiscountComponent não precisa ser alterado para suportar novas regras — basta criar novas implementações.
